# Rules

1. When any update is made, update README.md changelog accordingly.
2. create new branch for every new big plan
3. Micro-commit every individual change — one commit per single change, never batch multiple changes into one commit. we are trying to have as much commits as possible.
4. please commit as much as possible: in each small update, if there are few edits in each, commit all the few edits separately.
5. Never include Claude as an author or co-author in git commits.
